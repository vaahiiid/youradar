5. Multiple accounts per provider

The app must support multiple connected accounts per provider.

Users should be able to connect:
- multiple Gmail accounts
- multiple Outlook accounts
- multiple Yahoo Mail accounts
- multiple AOL Mail accounts
- multiple Hotmail accounts
- multiple Instagram accounts
- multiple Facebook accounts/pages
- multiple LinkedIn accounts/pages where official API allows
- multiple Telegram bots/chats where supported
- multiple WhatsApp Business numbers where supported
- multiple TikTok accounts where supported
- multiple X accounts where supported
- multiple delivery tracking items per courier

Important:
Do not enforce one account per provider.

Database and UI must support:
- one user can have many connected accounts
- each connected account belongs to one provider
- each provider can have many connected accounts
- each connected account has its own notification settings
- each connected account has its own connection status
- each connected account has its own last sync/webhook time
- notifications must link to the exact connected account that generated them

6. Email providers

Add Yahoo Mail, AOL Mail, and Hotmail.

Email provider handling:
- Gmail should use Gmail API.
- Outlook and Hotmail should use Microsoft Graph where supported.
- Yahoo Mail and AOL Mail should use secure OAuth/IMAP where available, but only if secure and allowed.
- Do not ask users for raw email passwords.
- If IMAP is used, prefer OAuth where supported.
- If a provider cannot support secure notification webhooks, implement a clearly labelled fallback sync option.

Important:
The product should explain that real-time notifications depend on provider API support.

Provider cards to add:
- Yahoo Mail
- AOL Mail
- Hotmail

7. Connected provider visibility rule

Do not show all connected account details before users connect them.

UI rule:
- In the main dashboard, only show providers/accounts that the user has connected.
- In notification filters, show only connected providers plus “All”.
- In account badge sections, show only connected accounts.
- In the Connected Accounts page, it is okay to show available providers as “Connect” options.
- Once a user connects a provider/account, show it in their dashboard and notification filters.
- If they disconnect it, hide it from the main dashboard and filters unless there are historical notifications that need a filter.
