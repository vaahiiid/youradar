Update the existing YourRadar app design and integrations.

Do not rebuild the app from scratch. Continue from the current version.

1. Background and colour update

Change the main app background to white / off-white.

The current dark-heavy design should be softened. The app should feel clean, premium, mobile-first, and modern.

Use a colour palette that matches the attached logo.

Requirements:
- Main background: white or off-white
- Cards: white with soft shadow
- Text: dark navy / charcoal
- Radar elements: bold and clearly visible
- Accent colours should be taken from or inspired by the attached logo
- Remove any colours that clash with the logo
- Keep notification badges red for new/unread alerts

Suggested palette:
- Background: #FFFFFF
- Soft background: #F7F9FC
- Primary text: #0B1020
- Secondary text: #667085
- Border: #E5E7EB
- Notification red: #FF3B30
- Use the logo colours for primary buttons, highlights, and radar effects

2. Make radar elements stronger

The radar concept should be more visible and more central to the design.

Use bold radar elements across the app:
- Startup loading screen
- Dashboard hero area
- Notification centre
- Empty states
- Loading states
- Account connection cards
- Provider cards
- Background decorative patterns
- Mobile navigation selected state

Radar design should include:
- bold circular radar rings
- clear scan lines
- animated sweep effect
- signal dots
- pulsing notification points
- subtle radar grid patterns
- radar-style loading indicators

Do not make the radar elements too faint. They should be part of the brand identity.

3. Loading system

All loading states should use radar-themed loaders.

Replace generic spinners with:
- full-page radar loader with logo in the centre
- mini radar loader for buttons
- radar scan skeletons for dashboard cards
- “Scanning your signals...” loading text where appropriate

6. Important API limitation rules

Do not use unofficial scraping, password login, browser bots, private APIs, or anything that violates platform terms.

Each provider must be designed around official APIs only.

The app should clearly explain that some integrations may have limitations depending on official API access, account type, permissions, and app review approval.

10. Provider-specific technical direction

LinkedIn:
- Design for official LinkedIn APIs only.
- Do not scrape LinkedIn.
- Add placeholder architecture for LinkedIn notifications where API permissions are available.

Facebook:
- Use Meta Graph API and Webhooks where available.
- Support Facebook Page-related events where permissions allow.
- Do not scrape personal Facebook notifications.

Telegram:
- Support Telegram Bot API integration.
- Allow users to connect a Telegram bot or channel/chat where technically possible.
- Use webhooks for Telegram bot updates.

WhatsApp:
- Use official WhatsApp Business Cloud API only.
- Support business message notifications where available.
- Do not attempt to read personal WhatsApp messages or personal WhatsApp app notifications.

TikTok:
- Use official TikTok developer APIs only.
- Add placeholder support for supported business/creator events where API access allows.
- Do not scrape TikTok notifications.

11. Trust and privacy copy

Add a privacy explanation near the Connected Accounts page:

“YourRadar only uses official provider APIs and only requests the minimum permissions needed to show supported notifications. We do not ask for your social media passwords, and we do not scrape your private accounts. Some platforms limit what can be monitored depending on account type, permissions, and app approval.”

- The attached logo is used properly.

- The notification filters include all providers.
