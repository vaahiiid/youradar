Fix the UI/UX issues in the current YourRadar mobile screens.

Do not rebuild the app from scratch. Only fix the current UI.

The “Simulate signal” section is only for testing. It should not appear in the normal user dashboard.

Change it as follows:
- Hide “Simulate signal” by default in production/normal mode.
- Move all mock/simulate notification buttons to Settings > Developer / Test Mode.
- Add a toggle called “Enable test mode” in Settings.
- Only show simulation buttons when test mode is enabled.
- The main dashboard should feel like a real user app, not a demo/testing page.

2. Dashboard layout cleanup

On the My Radar dashboard:
- Keep the logo and “My Radar” heading.
- Keep the Active Signals radar card.
- Keep connected source cards.
- Remove test/demo UI from the main dashboard.
- Make the dashboard more polished and less crowded.
- Make sure all horizontal carousels have proper padding and do not cut content.
- Make sure nothing overflows outside the phone screen width.

3. Fix Alerts page horizontal overflow

The Alerts page currently has serious mobile UI issues:
- Filter tabs overflow horizontally.
- The Instagram tab is cut off.
- Alert cards are too wide and touch the screen edges.
- Some elements are clipped.
- Bottom content is hidden behind the bottom navigation.

Fix this properly.

Requirements:
- The whole Alerts screen must fit inside mobile width.
- No horizontal overflow anywhere.
- Add proper left/right padding, around 16px to 20px.
- Filter tabs should be horizontally scrollable with smooth scrolling.
- Filter tabs should not be clipped.
- Add padding-right at the end of the filter row so the last tab is fully visible.
- Alert cards should use width: 100% within the padded container.
- Alert cards should not exceed the viewport width.
- Use box-sizing: border-box globally.
- Add enough bottom padding so the last alert card is not hidden behind the bottom nav.
